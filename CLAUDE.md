# CLAUDE.md - my-trip-note ブログプロジェクト

## プロジェクト概要

一人旅・海外旅行ブログ「my trip note」。
AstroPaper（Astro製）+ Cloudflare Pages で構築。
AmebaOwndからの記事をリライトして投稿。

## 著者プロフィール

- 20代後半に会社員を辞めて世界一人旅に出た。
- 当時の旅行記をリライトして投稿している。
- 現在は日本でサラリーマンとして働きながらブログを運営。

## 記事執筆ルール

- 旅の記事は**過去形**で書く（当時29歳の体験談として）
- 「あの頃は〜だった」「今思えば〜」など、39歳の現在から振り返る視点を自然に盛り込む
- 持ち物・アプリ・サービスなどの情報は**2026年時点の最新情報**にアップデートする
- 古いサービス名や廃止されたアプリは現在の代替サービスに置き換える
- 過去の体験と現在の知見を組み合わせた「リアルなアドバイス」として書く
- **frontmatter の `pubDatetime` は実際の投稿日時を設定する**（過去の旅行記をリライトしても、投稿日は「今日」の日付を使用）

## 画像ルール

- **記事ごとに異なる画像を使うこと。同じ画像を複数記事で使い回し禁止。**
- 既存の画像（`public/images/` 配下）と重複しないよう必ず確認する
- 適切な画像がない場合は、以下のフリー素材サイトからダウンロードして使う
  - [Unsplash](https://unsplash.com/)（商用利用可・クレジット不要）
  - [Pexels](https://www.pexels.com/)（商用利用可・クレジット不要）
  - [Pixabay](https://pixabay.com/)（商用利用可・クレジット不要）
- ダウンロードしたファイルは `public/images/` に保存し、ファイル名は内容がわかる英語名にする（例：`travel-apps-smartphone.jpg`）
- アプリアイコン・企業ロゴ・App Storeのスクリーンショットは著作権の観点から使用しない

## Kensukei の書き癖・文体メモ

記事を書くときやリライトするときは、以下の文体・スタイルを再現すること。

- **基本トーン**：ですます調と口語が混在。「〜のだ」「〜である」で断言。読者を旅仲間として扱う
- **テンポ**：短文と長文が交互。句点を多く打つ。1文は1〜2行程度
- **「〜のだ」「〜のである」**：強調・締め・断言の場面のみ。連発すると説教くさくなるので注意
- **感嘆符（！）**：本当に興奮・驚いた場面に絞る。説明文は避ける
- **「笑」「・・・・」**：セルフツッコミや余韻の表現として使用
- **表現**：「かなり」「非常に」「圧倒的な」など強調副詞を多用。「是非〜してほしい」「参考にしてほしい」など推薦フレーズも多い
- **構成**：導入で短文で勢いよく描写 → 本文で話題転換 → 「つづく」か「〜である！」で力強く締める
- **個性**：自虐ユーモア、ピンチをポジティブに楽しむ、「人生一度きり」「後悔しない」といった座右の銘

## 記事管理

- 記事ファイル: `src/data/blog/` 配下に `.md` ファイルで管理
- 画像: `public/images/` 配下
- GitHubにpushすると自動でCloudflare Pagesにデプロイされる

## 作業フロー（必須ルール）

### 記事作成前の自動確認フロー（必須）

「リメイク」「投稿」「記事作成」指示を受けたら、**ユーザーに確認なしに以下を自動実行**：

1. **既存画像との重複確認**
   - `ls public/images/` で現在のファイル一覧を取得
   - 記事テーマに合わせて新しい英語ファイル名を決定（重複なし）
   - 同じファイル名を複数記事で使わない

2. **frontmatter形式を自動設定**
   - `title`: SEOキーワード + 「｜」 + サブタイトル形式
   - `description`: 具体的な体験・数字を含む説明文
   - `pubDatetime`: 本日の日付を `YYYY-MM-DDT00:00:00Z` 形式で自動設定
   - `tags`: 関連キーワードのリスト
   - `featured: false`, `draft: false` は常に固定

3. **ファイルパス構造を確認**
   - 記事は `src/data/blog/[英語スラッグ].md` 形式で保存
   - スラッグは kebab-case（例：`chatuchak-night-market.md`）

4. **ユーザー提供画像の処理**
   - ユーザーが「写真はそのまま使ってほしい」と言った場合：
     - Ameba Ownd等のエディタから写真を自動取得
     - `public/images/` に英語ファイル名で保存
     - PDFの場合は `sips -s format jpeg` で自動変換

---

### 記事作成・修正後の確認フロー

1. **MDファイル全文をユーザーに表示して確認依頼を出す（必須）**
   - `Read` ツールで `src/data/blog/[スラッグ].md` を表示
   - frontmatterから末尾まで全文表示。要約・抜粋は禁止
   - ユーザーの承認（「いいよ」「OK」等）を得てからコミット

2. **コミット＆プッシュ後のWebページ確認（必須）**
   - 90秒待機（Cloudflare Pages デプロイ完了を待つ）
   - `curl` でHTTPステータス（200）を確認
   - Claude-in-Chrome でページを開きスクリーンショット確認
   - 画像・レイアウト・本文が正常に表示されているかチェック
   - 問題があれば即修正して再プッシュ

## パッケージ管理ルール

### pnpm-lock.yaml と package.json の同期（重要）

**⚠️ package.json を編集・追加したら、必ず以下を実行してから commit すること：**

1. `pnpm install` を実行して `pnpm-lock.yaml` を最新化する
2. `git status` で `pnpm-lock.yaml` に変更がないか確認する
3. package.json と pnpm-lock.yaml を一緒に commit・push する

**理由：** Cloudflare Pages は pnpm で厳密にチェックするため、lockfile と package.json が不整合だとデプロイが失敗する（2026-05-15に発生）。詳細は memory/error_pnpm_lockfile.md 参照。

**最も安全な方法：** 本プロジェクトの dependencies を更新する場合は、ローカルで pnpm を使用することで、常に lockfile が自動同期される。
